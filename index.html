Running Claude Code — the must-knows

You don't need most of this to start. The top five are the must-knows; the rest is there for when you want it. You're on the Pro plan.

Three levels below — only the first is a must. Everything under it is there when you want it, not homework.

★ Must-readthe five things to actually know
1Start it

Open a terminal in your ~/family-assistant folder and type claude. That's it.

First session ever: /tour. Every day after: /daily-open.

2Pick the model

Type /model to switch any time:

Sonnet 5your default — fast, smart enough for ~95% of things. Start here.
Opus 4.8when Sonnet gets stuck, or for genuinely hard reasoning. Slower, uses your quota faster.
Haiku 4.5light and fast — quick edits, simple lookups. Easiest on quota.
Fable 5the flagship, for the very hardest long jobs. Rarely needed; burns quota fastest.
the trade-off, plainly
Smarter model = better on hard problems but slower and heavier on your limit (see #5). Stay on Sonnet; reach for Opus/Fable only when a problem needs it, then switch back.
3Choose how careful

Press Shift+Tab to cycle how much it asks before acting:

Normalasks before changing files. Safe default.
Auto-acceptstops asking, just makes the edits. Faster.
Plan-firstlays out a plan and waits for your OK. Good for bigger tasks.
which to use
Start in Normal. Use Plan-first to see the approach before it runs; Auto-accept once you trust it. (A hands-off "auto" mode may also appear in the cycle.)
4Watch the context

Each session has a memory ("context"). When it fills, quality drops. Your status line shows the model and context left — Reut sets that up.

/contextsee how full it is now.
/compactsqueeze the history to keep going on the same topic.
/clearwipe and start fresh — do this when you switch to an unrelated task.
when to bother
Mostly you don't — it auto-compacts around 80% full. The one good habit: /clear between unrelated tasks. Your notes are saved to files regardless, so clearing loses nothing important.
5Mind the limits

Pro gives you two usage windows: a rolling 5-hour one and a weekly one. Type /usage to see where you stand.

Heavier models eat the limit faster — Opus ~1.7×, Fable ~3.3× Sonnet's rate. Opus also has its own separate cap: hit "Opus limit" and Sonnet/Haiku still work.

If you run out: wait for the window to reset, or switch to a lighter model (/model → Sonnet or Haiku). Staying on Sonnet is the easiest way not to hit limits. (Max plans have bigger windows if you ever outgrow Pro.)
◐ Good to knowhandy — dip in when you want
Hand off & resume

Stopping mid-task, or the context is filling up? Don't push a tired session — have it write a handover: a short note of where things stand and what's next. A fresh session reads it and picks up cleanly.

Ask for one"write a handover before we stop" → it saves the state + next steps to a note.
/parka lighter version — stash one thread with enough context to resume.
Next time"read the handover and continue," or /daily-open then point it at the note.
when to hand off
Three moments: (1) you hit a natural stop with unfinished work; (2) /context is getting full — a fresh session + a handover beats a bloated one; (3) you want a different model for the next stretch. The handover is just a note in your vault, so it survives a /clear or a new day.
Ask, don't dig

Forget something? Ask in plain words: what can you do?, how do I…?. It answers. You never memorize commands.

Subagents helpers

For a big or side task, it can send off a helper agent that works separately and brings back just the answer — keeping your main chat clean.

when it helps
"Go read all these files and summarize," or "look into X while we keep going." You don't set it up — just ask; it decides.
○ If you're curiouspower tools — entirely optional, none needed to work
Workflows advanced

For a large job it can run many agents at once in a structured way, then pull the results together.

the catch
Powerful but heavy on quota (many agents at once), so it's opt-in — ask for it when you have something genuinely big.
Firecrawl web

Reads live web pages. Hand it a link and it pulls that page's current contents in cleanly — more than a plain URL gives you. Good for up-to-date info from a specific site.

status & use
Reut is setting this up. Once it's in, "read this page and pull the numbers" works on a link you give it. For open-ended "search the web," that's the coming Perplexity integration — different thing.
Context Mode big jobs

A sandbox for heavy lifting. When there's a lot to chew through (long logs, big files, piles of data), it works off to the side and brings back just the answer — keeping your conversation clean. You have this; it mostly runs on its own.

when it kicks in
Automatic for large outputs — you don't invoke it. The upshot: big analysis jobs don't fill up your context or slow the chat down.
